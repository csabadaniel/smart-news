# Copilot Instructions for smart-news

## Commit Convention

When creating commits, always follow the **Conventional Commits** format defined in [CONTRIBUTING.md](CONTRIBUTING.md).

**Quick reference:**
- Format: `<type>: <subject>`
- Use lowercase type prefix (feat, fix, ci, docs, refactor, test, build, chore)
- Imperative mood: "add" not "added"
- No period at end
- Example: `ci: add workflow_dispatch trigger`

For complete guidelines, see [CONTRIBUTING.md](CONTRIBUTING.md).
